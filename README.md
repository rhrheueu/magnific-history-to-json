# Freepik History to JSON

Chrome-расширение для выгрузки истории промптов Freepik (Pikaso) в JSON для отчётов.

## Что делает
- Проматывает ленту проекта в `https://www.freepik.com/pikaso/projects/` до конца.
- Работает на:
  - `https://www.freepik.com/pikaso/projects/home`
  - `https://www.freepik.com/pikaso/projects/<project-id>`
  - `https://www.freepik.com/pikaso/projects/history`
- Собирает данные по каждому промпту:
  - `prompt_index` (порядковый номер промпта в выгрузке)
  - `prompt` (текст промпта)
  - `date_group` (текстовая группа даты из интерфейса: `Today`, `This week`, `February` и т.п.)
  - `model` (название модели)
  - `quality` (например, `auto`)
  - `types` (тип генерации, например `text to image`)
  - `resolutions` (например, `2K`)
  - `tags` (полезные теги, без служебных)
  - `images` (массив превью с `url` и `file` для связи JSON ↔ файлы)
- Сохраняет JSON файл через стандартную загрузку Chrome.
- Опционально скачивает превью-картинки в папку вида `freepik_history_YYYY-MM-DD_previews/`.
- Есть режимы выгрузки:
  - `С картинками` (по умолчанию)
  - `Только текст` (без скачивания превью)
- Перед выгрузкой с картинками показывает предупреждение, что процесс может занять несколько минут.

## Установка
1. Открой `chrome://extensions`.
2. Включи `Developer mode`.
3. Нажми `Load unpacked` и выбери эту папку.
4. Закрепи иконку расширения (через меню пазлика), чтобы она была видна.

## Использование
1. Открой любую страницу проекта:
   - `https://www.freepik.com/pikaso/projects/home` (Personal project),
   - или `https://www.freepik.com/pikaso/projects/<project-id>`.
2. Если нужно — выстави фильтры/даты вручную.
3. Нажми на иконку расширения.
4. Дождись окончания прокрутки (появится pop-up с прогрессом).
5. Если нужно завершить раньше — нажми `Stop` (файл всё равно будет сформирован).

## Режим выгрузки
- По умолчанию: `С картинками`.
- Быстрое переключение: правый клик по иконке расширения → `Режим выгрузки`.
- Через настройки: правый клик по иконке расширения → `Параметры` (Options).
- В режиме `С картинками` перед стартом появляется предупреждение:
  - `OK` — продолжить с картинками.
  - `Отмена` — полностью отменить текущую выгрузку (без запуска парсинга).
  - Чтобы включить `Только текст`, переключи режим через контекстное меню и/или `Параметры` (Options).

## Имя файла
- Имя выгрузки теперь включает аккаунт, проект и время до минуты:
  - `freepik_history_<account>_<project>[_<projectId8>]_<dd.mm.yy.hh.mm>.json`
- Пример:
  - `freepik_history_personal_project_work_efbfcded_03.03.26.20.20.json`

## Если страница подгружается медленно
Увеличь лимит проверок конца ленты в `parser.js`:

- `parser.js`
- Настройка: `SETTINGS.endCheckCycles` (по умолчанию `60`).
- Для выгрузки превью на уровне кода: `SETTINGS.exportPreviewImages` (`true`/`false`).

Чем больше значение, тем дольше скрипт будет «дожидаться» новых элементов перед остановкой.

## Структура
- `manifest.json` — манифест MV3.
- `background.js` — инжект скрипта и скачивание JSON.
- `parser.js` — логика парсинга + UI прогресса.
- `icons/` — иконки расширения.

## Публикация в Git
- Скрипт: `scripts/publish_git.sh`
- Быстрый publish (коммит + push текущей ветки):
  - `scripts/publish_git.sh -m "feat: update exporter"`
- Если нужно включить новые файлы (untracked):
  - `scripts/publish_git.sh -m "feat: add files" --all`
- Только коммит без push:
  - `scripts/publish_git.sh -m "wip: local commit" --no-push`
